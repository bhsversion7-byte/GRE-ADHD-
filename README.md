# GRE 图像记忆单词系统（适合 ADHD 等）

这是一个正在构建中的 GRE 词汇与 verbal 训练网站。项目目标不是做普通背单词工具，而是为注意力容易漂移、工作记忆负担较高、前额叶执行功能较弱，或需要更强视觉线索的学习者，设计一套更容易进入状态、持续复习、减少挫败感的学习系统。

## 项目初衷

传统 GRE 背词通常依赖大量重复、抽象释义和长时间自控。对 ADHD、注意力调节困难、前额叶较薄或执行功能压力较大的人来说，这种方式容易出现几个问题：

- 一次要处理的信息太多，开始之前就已经消耗精力。
- 单词释义抽象，缺少可抓住的画面和情境。
- 做题和背词记录分散，错题、忘词和复习计划不能自然联动。
- 刷新页面或中断学习后进度丢失，重新进入成本高。
- 传统界面反馈弱，难以形成“我正在推进”的感觉。

因此，这个项目把 GRE 单词、视觉记忆、主动回忆、错题库、间隔复习和用户笔记放在同一个网站里，用小块、清晰、有反馈的方式帮助用户完成高强度学习。

## 当前状态

项目仍在开发中，目前尚未开放公网网站，也不是稳定版本。当前主要是本地可运行的 Next.js 网站，会持续更新词库、题库、视觉设计、错题系统和未来部署方案。

后续计划包括：

- 部署到 Cloudflare Pages 或其他公网访问平台。
- 完善用户中心、云端同步和多设备访问。
- 继续提升 GRE 填空、阅读、同义词等题型的解析质量。
- 更细致地支持每日 200-500 个单词的学习节奏。
- 扩充本地图像资产，减少对外部 API key 的依赖。

## 如何访问

这个项目最终应该以“打开网页即可使用”为主要方式，而不是要求每个用户都在本地安装开发环境。

### 普通用户

普通用户不建议直接下载源码运行。当前公网网站还没有开放；开放后，README 顶部会放置正式访问地址。到时候用户只需要打开网站即可使用单词卡、测试、错题库和本地图像记忆功能。

### 开发者本地运行

如果你是开发者，或想提前体验本地版本，可以使用下面的方式。

环境要求：

- Node.js 22 或更新版本
- pnpm 11.5.2
- Git

macOS / Linux / Windows 通用命令：

```bash
git clone https://github.com/bhsversion7-byte/GRE-ADHD-.git
cd GRE-ADHD-
corepack enable
corepack prepare pnpm@11.5.2 --activate
pnpm install
pnpm dev
```

启动后打开：

```bash
http://localhost:3000
```

Windows 用户也可以直接运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/start-windows.ps1
```

这个脚本会检查 Node.js、准备 pnpm、安装依赖并启动本地开发服务。

### 部署给所有人访问

推荐下一步部署到 Cloudflare Pages 或 Vercel。部署完成后，GitHub 仓库主要作为开发者入口；真实用户入口应该是公网 URL。

Cloudflare Pages 的建议配置：

- Build command: `pnpm install && pnpm build`
- Output / runtime: 使用当前 Next.js + OpenNext 配置
- Node version: 22 或更新版本

Vercel 的建议配置：

- Framework preset: Next.js
- Install command: `pnpm install`
- Build command: `pnpm build`

目前仓库不包含用户上传的原始 PDF 和答案文件，只包含代码、处理后的本地 JSON 数据和本地图像资产。

## 已实现功能

### 1. 每日学习目标

- 支持高强度 GRE 学习节奏，默认以每日 200-500 词为设计方向。
- 自动把学习拆成小组，避免一次面对过大的任务。
- 学习进度保存在本地，刷新后不会丢失当前位置。

### 2. 图像记忆单词卡

每个单词卡集中展示：

- 单词本身
- 音标
- GRE 常见中英文意思
- 例句
- 同义词
- 反义词
- 词根/词缀线索
- 视觉记忆场景
- 本地视觉图片
- 用户自己的记忆区/评论区

单词卡支持 pin 标记。用户可以手动标记特别容易忘的单词；如果在复习时选择“完全不记得”，系统也会自动同步到毫无印象单词库。

### 3. 本地图像资产

当前项目使用 `public/asset/word-images/` 中的本地 SVG 图像资产作为单词视觉图，不需要 API key 即可展示。之后如果配置 AI 图片生成，本地资产仍会作为 fallback。

### 4. 主动回忆测试

测试页按 GRE verbal 结构设计，不是简单的“看中文猜英文”。

目前支持：

- 单空填空题
- 多空填空题
- 同义词等价题
- 阅读 passage 题
- 拖拽/点击选项进入空格
- 提交后显示对错、正确答案和解析
- 做题结果自动保存
- 刷新后恢复当前题目和题库模式

测试页左侧有题目预览：

- 已做且正确：淡绿色
- 已做且错误：淡红色
- 未做：透明底黑字
- 当前题：深色边框
- 点击数字圈可立即跳转

### 5. 错词与错题系统

错词区分为两个部分：

- 题目错误库：做错的题目会进入这里。点进去重新做对后，可以从错题库移出；如果持续做错，会记录 pin 次数。
- 毫无印象单词库：与单词卡片中的“完全不记得”同步。

### 6. 用户笔记与导出

- 每个单词都有记忆区/评论区，便于写联想、错因、谐音和例句。
- 不会的单词支持导出为 Excel 可打开的格式，方便后续用 Microsoft Excel 管理。

### 7. 深色模式与低干扰 UX

- 支持 dark mode 切换。
- 视觉风格尽量保持简洁、清晰、低干扰。
- 通过小反馈、进度保存和 celebration 动效帮助用户维持学习动量。
