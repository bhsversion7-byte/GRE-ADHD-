# GRE 图像记忆单词系统（适合 ADHD 等）

这是一个正在构建中的 GRE 词汇与 verbal 训练网站。项目目标不是做普通背单词工具，而是为注意力容易漂移、工作记忆负担较高、前额叶执行功能较弱，或需要更强视觉线索的学习者，设计一套更容易进入状态、持续复习、减少挫败感的学习系统。

## 当前状态

项目目前采用本地开发服务器体验，不使用 GitHub Pages，也不维护 `index_1.html` 静态核对页。完整产品请通过本地 `localhost:3000` 打开。

当前仓库包含：

- Next.js / React / TypeScript 前端项目
- 本地 GRE 词库与题库 JSON
- 本地图像资产与开放许可图片 manifest
- 学习、视觉、测试、错词、词根、用户、导入、设置等完整功能区

仓库不包含用户上传的原始 PDF 和答案文件。

## 项目初衷

传统 GRE 背词通常依赖大量重复、抽象释义和长时间自控。对 ADHD、注意力调节困难、前额叶较薄或执行功能压力较大的人来说，这种方式容易出现几个问题：

- 一次要处理的信息太多，开始之前就已经消耗精力。
- 单词释义抽象，缺少可抓住的画面和情境。
- 做题和背词记录分散，错题、忘词和复习计划不能自然联动。
- 刷新页面或中断学习后进度丢失，重新进入成本高。
- 传统界面反馈弱，难以形成“我正在推进”的感觉。

因此，这个项目把 GRE 单词、视觉记忆、主动回忆、错题库、间隔复习和用户笔记放在同一个网站里，用小块、清晰、有反馈的方式帮助用户完成高强度学习。

## 如何安装和打开 localhost:3000

### 环境要求

- Git
- Node.js 22 或更新版本
- pnpm 11.5.2

如果电脑已经安装 Node.js 22，可以直接用 Corepack 准备 pnpm。

### 第一次安装

```bash
git clone https://github.com/bhsversion7-byte/GRE-ADHD-.git
cd GRE-ADHD-
corepack enable
corepack prepare pnpm@11.5.2 --activate
pnpm install
pnpm dev
```

启动成功后，在浏览器打开：

```text
http://localhost:3000
```

### Windows 快速启动

Windows 用户也可以在项目目录运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/start-windows.ps1
```

这个脚本会检查 Node.js、准备 pnpm、安装依赖并启动本地开发服务器。启动后同样打开：

```text
http://localhost:3000
```

### 每次回来继续使用

如果依赖已经安装过，通常只需要：

```bash
cd GRE-ADHD-
pnpm dev
```

然后打开：

```text
http://localhost:3000
```

### 常见问题

- 如果 `localhost:3000` 打不开，先确认终端里 `pnpm dev` 还在运行。
- 如果 3000 端口被占用，Next.js 可能会提示使用 3001；以终端显示的地址为准。
- 学习进度、错题、pin 标记和笔记保存在当前浏览器的 `localStorage` 中。换浏览器或清理浏览器数据后，本地记录会变化。
- 如果开发模式弹出 hydration warning，并且提示里出现 `data-new-gr-c-s-check-loaded` 或 `data-gr-ext-installed`，通常是 Grammarly 等浏览器插件在 React 加载前修改了页面属性。这个问题不影响学习功能；项目已在根布局中加入兼容处理。如果仍出现，可以临时关闭相关浏览器插件或用无痕窗口打开 `http://localhost:3000`。
- 需要完整交互时必须使用 `pnpm dev` 启动本地站点；仓库已删除 `index_1.html`，不再支持双击 HTML 作为正式查看方式。

## 已实现功能

### 1. 每日学习目标

- 支持高强度 GRE 学习节奏，默认以每日 200-500 词为设计方向。
- 自动把学习拆成小组，避免一次面对过大的任务。
- 学习进度保存在本地，刷新后不会丢失当前位置。

### 2. 图像记忆单词卡

每个单词卡集中展示：

- 单词本身
- 音标
- GRE 常见中英文意思
- 例句和中文翻译
- 同义词
- 反义词
- 词根/词缀线索
- 视觉记忆场景与图片
- 用户自己的记忆区/评论区

单词卡支持 pin 标记。用户可以手动标记特别容易忘的单词；如果在复习时选择“完全不记得”，系统也会自动同步到毫无印象单词库。

### 3. 主动回忆测试

测试页按 GRE verbal 结构设计，目前支持：

- 单空填空题
- 多空填空题
- 同义词等价题
- 阅读 passage 题
- 拖拽/点击选项进入空格
- 提交后显示对错、正确答案和解析
- 做题结果自动保存
- 刷新后恢复当前题目和题库模式

### 4. 错词与错题系统

错词区分为两个部分：

- 题目错误库：做错的题目会进入这里。重新做对后，可以从错题库移出；如果持续做错，会记录 pin 次数。
- 毫无印象单词库：与单词卡片中的“完全不记得”同步，只显示英文和中文意思，点击后跳转到对应视觉卡。

### 5. 用户笔记与导出

- 每个单词都有记忆区/评论区，便于写联想、错因、谐音和例句。
- 不会的单词支持导出为 Excel 可打开的格式，方便后续用 Microsoft Excel 管理。

### 6. 深色模式与低干扰 UX

- 支持 dark mode 切换。
- 视觉风格尽量保持简洁、清晰、低干扰。
- 通过小反馈、进度保存和 celebration 动效帮助用户维持学习动量。
