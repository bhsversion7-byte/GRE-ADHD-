# GRE 图像记忆单词系统（适合 ADHD 等）

这是一个面向 GRE 高频背词、填空训练和阅读机经复盘的学习网站。它尤其考虑 ADHD、注意力容易漂移、执行功能压力较大、前额叶调控较弱，或需要更强视觉线索的学习者。

项目仍在构建中，暂时还没有正式开放公网网站。当前仓库包含可运行的 Next.js 项目、本地词库/题库 JSON、本地开放许可图片资产，以及一个不需要启动开发服务器的静态预览文件。

## 项目初衷

很多 GRE 背词工具把重点放在“重复次数”和“词义列表”，但对注意力调节困难的人来说，真正困难的是：

- 一次看到的信息太多，很难开始。
- 抽象词义没有画面，记忆抓不住。
- 背词、做题、错题、笔记分散，复习线索断掉。
- 页面刷新或中断后进度丢失，重新进入状态成本高。
- 每天 200-500 个词的强度下，普通卡片很容易让人疲劳。

这个项目的目标是把单词、图片、中文释义、英文释义、例句、同义词、反义词、词根、笔记、错题和复习节奏放到同一个学习系统里，尽量减少切换成本。

## 现在怎么体验

### 方式 1：直接打开静态预览文件

如果只是想核对内容、看词库、看图片是否对应、检查阅读布局，不需要启动本地开发服务器。

直接打开仓库根目录里的：

```text
index_1.html
```

这个文件可以直接双击打开。它内置了当前词库和阅读题库的精简数据，并引用 `public/asset/word-photos/` 与 `public/asset/word-images/` 里的本地图片。

适合检查：

- 词库是否导入完整
- 单词图片是否明显不匹配
- 填空题干、选项、答案是否能读
- 阅读是否是左边编号、中间文章、右边题目的三栏布局
- 毫无印象单词库是否足够紧凑

注意：`index_1.html` 是内容核对页，不是正式产品。正式学习体验仍以 Next.js 网站为准。

### 方式 2：本地运行完整网站

适合开发者或想体验完整交互的人。

环境要求：

- Node.js 22 或更新版本
- pnpm 11.5.2
- Git

通用命令：

```bash
git clone https://github.com/bhsversion7-byte/GRE-ADHD-.git
cd GRE-ADHD-
corepack enable
corepack prepare pnpm@11.5.2 --activate
pnpm install
pnpm dev
```

然后打开：

```text
http://localhost:3000
```

Windows 用户也可以运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/start-windows.ps1
```

### 方式 3：未来公网网站

普通用户最终不应该下载源码或安装开发环境。这个项目后续会部署到 Cloudflare Pages、Vercel 或类似平台。到时候用户只需要打开一个公网网址，就可以使用单词卡、测试、错题库、笔记和图片记忆功能。

## 当前功能

### 图像记忆单词卡

单词卡展示：

- 英文单词
- 音标
- GRE 常见中英文意思
- 例句和中文翻译
- 同义词
- 反义词
- 词根/词缀线索
- 视觉记忆图片
- 记忆钩子
- 用户笔记区

当前图片优先使用 `public/asset/word-photos/` 中的开放许可真实图片；没有真实图片时，会回退到 `public/asset/word-images/` 中的本地 SVG 记忆图。

### 每日学习

项目以每日 200-500 词的高强度学习为设计背景，把学习拆成小轮次，尽量降低开始和继续的心理成本。

### 填空题库

填空题库保留 GRE 原题结构，包括：

- 单空题
- 多空题
- 同义词等价题
- 选项拖拽/点击进入空格
- 提交后显示对错
- 正确答案和解析区域
- 做错后进入题目错误库

### 阅读题库

阅读题库采用三栏布局：

- 左边：Passage 编号预览
- 中间：阅读文章
- 右边：题干和选项

这样可以减少上下滚动带来的注意力断裂，更接近正式做阅读时的核对流程。

### 错词急救区

错词区分为两类：

- 题目错误库：做错的填空和阅读会进入这里。之后做对可以移出。
- 毫无印象单词库：和单词卡中的“完全不记得”同步，只显示英文和中文意思。点击词条会跳转到对应视觉卡。

### 本地导出

不会的单词可以导出为 Excel 可打开的格式，方便后续复盘。

## 数据和版权说明

仓库不包含用户上传的原始 PDF 文件。当前保留的是处理后的本地 JSON 数据、项目代码和图片资产。

单词视觉图片来自开放许可来源，并在 `public/asset/word-photos/manifest.json` 中记录标题、作者、许可和来源。由于开放图库数据也可能存在元数据不完整的问题，后续仍需要继续人工筛查明显不匹配的图片。

## 当前限制

- 项目还没有正式公网部署。
- 图片匹配还在逐批补齐，部分抽象词仍需要人工筛选更合适的图片。
- 题库解析仍在持续优化，尤其是阅读解析和多空题解释。
- 用户系统目前偏本地状态，未来需要云同步。

## 技术栈

- Next.js
- React
- TypeScript
- Tailwind CSS
- shadcn/ui 风格组件
- 本地 JSON 词库与题库
- Openverse 开放许可图片资产

## 项目定位

这个项目不是简单的单词表，而是一个为高强度 GRE 学习设计的视觉记忆系统。目标是让用户更容易开始、更容易继续、更容易复盘，也更容易在大量抽象词中建立可回想的画面线索。
